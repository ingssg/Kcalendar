# Kcalendar 기획 정리

> 이 문서는 기획 + 디자인 확정 세션의 결론을 담은 것으로, 기술 스택 수립 및 구현의 시작점이다.
> 마지막 업데이트: 디자인 확정 후 (Stitch v2 기준)

---

## 서비스 한 줄 정의

> **"자연어로 기록하고 주간 흐름을 보는, 판단 없는 순수 칼로리 기록 서비스"**

---

## 핵심 원칙

- AI는 수치 계산 도구일 뿐. 판단, 평가, 코멘트 없음
- 낮은 입력 마찰이 최우선
- 정확한 의료/영양 분석이 아니라 일일 칼로리 흐름을 빠르게 파악하는 도구

---

## 핵심 문제 정의

기존 칼로리 앱(MyFitnessPal 등)은 정확성을 요구하는 구조라 입력 마찰이 크고 지속 사용이 어렵다.
→ 자연어로 대충 적어도 기록이 가능한 구조로 이 문제를 해결한다.

---

## 타겟 사용자

- 다이어트 의식은 있지만 기존 앱이 번거로워서 포기한 경험이 있는 20-40대 일반인
- 정확한 영양 분석보다 "오늘 대략 어떻게 먹었나"를 빠르게 파악하고 싶은 사람

---

## 저장 방식

- 기본: 로컬스토리지
- 기록이 쌓이면 "영구 보관하려면 로그인" 유도 (강제 아님)
- 로그인은 v2 이후

---

## 화면 구조 (MVP 확정)

### 총 4개 화면

```
① 온보딩        최초 1회, 프로필 입력
② 탭1           오늘의 기록 (메인)
③ 탭2           주간 기록
④ 날짜 상세     주간 기록에서 날짜 클릭 시
```

---

### ① 온보딩 (최초 1회)

- 필수 3개 항목만: **성별 / 키 / 몸무게**
- 이 3개로 하루 기준 칼로리(표준) 계산
- 키/몸무게 입력 시 기준 칼로리 실시간 미리보기
- 숫자 입력 스피너(상승/하락 버튼) 없음
- 이후 탭1로 진입

```
Kcalendar
오늘 나는 얼마나 먹었을까?
─────────────────
성별 선택 (버튼 2개: 남 / 여)
키 입력 (숫자, cm) — 우측 단위 숫자와 동일 높이 정렬
몸무게 입력 (숫자, kg) — 우측 단위 숫자와 동일 높이 정렬
─────────────────
하루 기준 칼로리   1,840 kcal  ← 초록 포인트 카드
─────────────────
[시작하기]
```

---

### ② 탭1: 오늘의 기록 (메인)

- 상단: 오늘 날짜 (예: 4월 18일 (금))
- **3단 요약 카드**: 기준 / 섭취 / 기준 대비 — 각 수치에 kcal 단위 표시
  - 카드 배경: 기준 이하 → 초록 5% 틴트 / 기준 초과 → 빨강 5% 틴트
  - 카드 하단: 섭취 프로그레스 바 + "기준의 xx% 섭취" 텍스트
- 자연어 입력창 + 기록하기 버튼
  - placeholder: "바나나 2개, 제육덮밥, 콜라 1캔…"
- AI 파싱 결과: 음식별 항목 리스트 + 각 칼로리 + ✏️ 수정 아이콘
  - ✏️ 클릭 → 해당 항목 kcal 직접 수정 (오늘만)
  - AI 추정 불가 입력 → "추정 불가" 회색 표시, 합산 제외, ✏️ 는 유지
- 자정 경계: 일어난 날 기준, 취침 전까지는 당일로 처리
- 하단 탭바: [오늘] [주간]

---

### ③ 탭2: 주간 기록

- 주 단위 이동 (이전/다음 주 네비게이션)
- 각 날짜 행:
  - 기록 있는 날: 좌측 컬러 액센트 바 (기준 이하 → 초록, 초과 → 빨강) + +/- kcal 수치
  - 기록 없는 날: 컬러 바 없음, "+0" 회색 표시 (빈 날 ≠ 안 먹음 구분)
  - 오늘: "오늘" 뱃지
- 날짜 클릭 → ④ 날짜 상세로 이동
- 하단 요약 카드 (비대칭 벤토): 주간 합계 kcal + 기록 일수 (x/7)
- 하단 탭바: [오늘] [주간]

---

### ④ 날짜 상세 (읽기 전용)

- 탭2에서 날짜 클릭 시 진입, ← 뒤로가기로 탭2 복귀
- ②와 동일한 3단 요약 카드 (기준 / 섭취 / 기준 대비) + 상태 반응형 배경
- 음식 항목 리스트 — ✏️ 없음 (수정 불가)
- "읽기 전용" 뱃지 없음 (수정 UI가 없는 것 자체로 읽기 전용 인지)
- 하단 탭바: [오늘] [주간]

## 확정된 디자인 시스템

> 디자인 컨셉명: **"The Objective Monolith"**
> 파일 위치: `stitch_kcalendar_ai_food_logger/`

### 컬러 토큰

| 용도 | 토큰 | 값 |
|---|---|---|
| 배경 (base) | `surface` | `#f8f9fa` |
| 카드 (primary) | `surface-container-lowest` | `#ffffff` |
| 섹션 구분 | `surface-container-low` | `#f3f4f5` |
| 호버/액티브 | `surface-container-high` | `#e7e8e9` |
| 기준 이하 (−) | `secondary` | `#1b6d24` |
| 기준 초과 (+) | `tertiary` | `#7d000c` |
| 주요 텍스트 | `on-surface` | `#191c1d` |
| 보조 텍스트 | `on-surface-variant` | `#474747` |
| 버튼/강조 | `primary` | `#000000` |

### 상태 반응형 색상 규칙

- 기준 이하 (섭취 < 기준): 요약 카드 배경 `rgba(27,109,36,0.05)`, 프로그레스 바 초록
- 기준 초과 (섭취 > 기준): 요약 카드 배경 `rgba(125,0,12,0.05)`, 프로그레스 바 빨강
- 수치 자체에만 색상 적용 — 배경/아이콘에 과한 색상 금지

### 타이포그래피

- **헤드라인/숫자**: Manrope (Bold, ExtraBold)
- **본문/레이블**: Inter (Regular, Medium)
- 숫자는 항상 크고 굵게, 레이블은 작고 letter-spacing 넓게

### 디자인 원칙 요약

- 경계선 금지 → 배경색 변화로 구분 (No-Line Rule)
- 구분선 금지 → 여백과 톤 변화로 리스트 구분 (No-Divider Rule)
- 그림자: `0 12px 32px rgba(25,28,29,0.04)` — ambient light 효과
- 애니메이션: linear 200ms, 바운스 금지
