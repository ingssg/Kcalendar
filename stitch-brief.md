# Kcalendar — Stitch 디자인 브리프

> 이 문서는 Stitch에게 넘길 와이어프레임 + 디자인 방향 + 레퍼런스 통합 브리프입니다.

---

## 1. 서비스 개요

**Kcalendar**는 자연어로 음식을 입력하면 AI가 칼로리를 파싱해주는 모바일 웹 앱입니다.

- **핵심 가치**: 판단 없는 순수 칼로리 기록. AI는 수치 계산 도구일 뿐이며, 평가·코멘트·피드백 없음
- **차별점**: MyFitnessPal처럼 정확성을 요구하지 않음 — "바나나 2개, 제육덮밥" 같은 자연어 입력으로 충분
- **타겟**: 다이어트 의식은 있지만 기존 앱이 번거로워 포기한 경험의 20–40대

---

## 2. 화면 구조 (MVP 4개)

```
① 온보딩       최초 1회 — 성별/키/몸무게 입력
② 탭1          오늘의 기록 (메인 화면)
③ 탭2          주간 기록
④ 날짜 상세    탭2에서 날짜 클릭 시 진입 (읽기 전용)
```

---

## 3. 각 화면 상세 스펙

### ① 온보딩
- 앱 이름 + 한 줄 설명
- 성별 선택 버튼 2개 (남 / 여) — 토글 형태
- 키 입력 필드 (cm)
- 몸무게 입력 필드 (kg)
- 입력값 기반 하루 기준 칼로리 실시간 미리보기 표시
- [시작하기] CTA 버튼

### ② 탭1: 오늘의 기록 (메인)
- 상단: 오늘 날짜 (예: 4월 18일 금)
- **3단 요약 카드**: 기준 칼로리 / 오늘 섭취 / 기준 대비 차이
  - 차이: 미만이면 녹색(−), 초과면 빨강(+)
- 자연어 입력창 (placeholder: "바나나 2개, 제육덮밥, 콜라…")
- [기록하기] 버튼
- 음식 항목 리스트: 음식명 / kcal / ✏ 수정 아이콘
  - ✏ 클릭 → 해당 항목 kcal 인라인 수정 가능 (오늘만)
  - AI 추정 불가 항목: "추정불가" 표시, 합산 제외
- 하단 탭바: [오늘의 기록] [주간 기록]

### ③ 탭2: 주간 기록
- 이전/다음 주 네비게이션 (< 4/12 – 4/18 >)
- 날짜별 리스트: 날짜 / 기준 대비 차이 kcal
  - 기록 있는 날: +/- kcal 색상 구분
  - 기록 없는 날: "기록없음 +0" 회색 표시 (빈 날 ≠ 안 먹음)
  - 오늘 날짜: 강조 표시 + "오늘" 뱃지
- 날짜 클릭 → ④ 날짜 상세로 이동
- 하단 주간 요약: 이번 주 합계 kcal + 기록한 날 수
- 하단 탭바

### ④ 날짜 상세 (읽기 전용)
- 탭2에서 진입. ← 뒤로 버튼
- ②와 동일한 레이아웃이지만 수정 불가 (✏ 없음)
- "읽기 전용" 뱃지 표시 권장

## 4. 디자인 방향

| 항목 | 방향 |
|---|---|
| 디바이스 | 모바일 웹 우선 (375px 기준), 데스크탑 호환 |
| 톤 | 클린, 미니멀, 중립적 — 판단하지 않는 도구 느낌 |
| 색상 | 무채색 기반. 포인트는 차이값 색상 2가지만 (−: 녹색계, +: 붉은계) |
| 타이포 | 가독성 우선. 수치는 크고 굵게. 레이블은 작고 흐리게 |
| 공백 | 여유 있는 패딩. 답답하지 않은 레이아웃 |

### ❌ 포함하지 말 것
- 도넛 차트, 매크로 분석 바, 영양소 그래프
- 스트릭 카운터, 뱃지, 게임화 요소
- AI 코멘트 영역, 격려 문구, 경고 메시지
- 과한 색상 (3색 이상 포인트 색 사용 금지)
- 이모지 평가 (👍👎 등)

## 5. 레퍼런스 앱별 참고 포인트

### MyFitnessPal — Today 탭
**참고할 것**: 기준/섭취/차이를 상단 3단 요약으로 크게 표시하는 구조
**참고하지 않을 것**: 매크로 분석 바, 도넛 차트, 과도한 정보 밀도

### Cal AI (Behance 레퍼런스)
**참고할 것**: AI 파싱 결과를 음식별 항목 리스트로 나열하는 UX. 깔끔한 입력→결과 플로우
**참고하지 않을 것**: 푸드 스캔 카메라 UI, 과도한 색상

### Habitify
**참고할 것**: 주간 뷰 날짜 칸 레이아웃. 날짜 클릭 → 상세 진입 패턴. 깔끔한 리스트 UI
**참고하지 않을 것**: 습관 추적 특화 UI, 스트릭 시각화

### Cronometer
**참고할 것**: 입력창 단순함 (검색 없이 텍스트만). 일관된 미니멀 UI. 수치 중심 레이아웃
**참고하지 않을 것**: 영양소 상세 분석, 복잡한 목표 설정 플로우

## 6. 와이어프레임 파일

첨부 파일: `wireframe.html` — 4개 화면 정적 와이어프레임

- 실제 인터랙션 없이 레이아웃 구조와 정보 계층만 표현
- 빨간 점선 박스 = 동작 주석 (실제 디자인에서 제거)
- 폰트/색상은 미확정 — 구조와 정보 배치만 참고

## 7. 기술 제약 (디자인 참고용)

- 저장: localStorage (v1 기준) → 서버 없음, 실시간 동기화 없음
- 로그인: v1 없음 (v2 이후 추가 예정) → 로그인 유도 UI 공간만 확보
- AI 응답: 텍스트 파싱 결과 → 로딩 상태(skeleton) 고려 필요
- 자정 기준: 일어난 날 기준 처리 → 날짜 전환 엣지케이스 없음 (v1)
